# Shusteri Automation

Система автоматизации генерации экспортных документов (Invoice, Specification, Packing List) для компании Shusteri.

## Возможности

- Генерация трёх документов: Commercial Invoice, Specification, Packing List
- Два режима вывода: три отдельных `.xlsx` или один файл с тремя листами
- Поддержка пресетов клиентов (Shoesberry, Bootstyle и др.)
- Два режима обработки: **загрузка контейнера** (пары) и **отправка грузов** (полупары)
- Автоматическое разделение товаров по категориям длины стельки (≤24 см / >24 см)
- Интеграция с системой **Честный знак** — добавление КМ/КИЗ кодов маркировки
- Добавление КИЗ кодов в **уже готовый файл** (без перегенерации документов)
- **Графический интерфейс** (CustomTkinter) и CLI-режим

## Установка

```bash
pip install -r requirements.txt
pip install customtkinter   # для GUI
```

Настройте пресеты клиентов в папке `presets/` и поместите входные Excel файлы в папку `input/`.

## Запуск

### GUI (рекомендуется)

```bash
python ui_run.py
```

### CLI

```bash
python run.py
```

## Структура проекта

```
shusteri-automation/
├── ui/                        # GUI интерфейс (CustomTkinter)
│   ├── app.py                 # Главное окно, сайдбар, переключение вкладок
│   └── views/
│       ├── generation.py      # Вкладка «Генерация»
│       ├── presets.py         # Вкладка «Пресеты»
│       ├── history.py         # Вкладка «История»
│       ├── log_view.py        # Вкладка «Лог»
│       └── _inject_dialog.py  # Диалог добавления КИЗ в готовый файл
├── src/
│   ├── generators/            # Генераторы документов (invoice, spec, packing list)
│   ├── formatters/            # Форматирование Excel
│   ├── models.py              # Модели данных (Pydantic)
│   ├── parser.py              # Парсер входных файлов (режим контейнер)
│   ├── shipment_parser.py     # Парсер входных файлов (режим отгрузка)
│   ├── processor.py           # Обработка данных (режим контейнер)
│   ├── shipment_processor.py  # Обработка данных (режим отгрузка)
│   └── kiz_injector.py        # Инъекция КИЗ кодов в готовый файл
├── presets/                   # YAML-пресеты клиентов
├── input/                     # Входные Excel файлы
├── output/                    # Готовые документы
├── выгрузка честный знак/     # Файлы выгрузки КМ кодов из ЧЗ
├── config.yaml                # Конфигурация (размеры, колонки)
├── run.py                     # CLI точка входа
└── ui_run.py                  # GUI точка входа
```

## Требования

- Python 3.10+
- pandas
- openpyxl
- rich
- pydantic
- PyYAML
- customtkinter (только для GUI)
